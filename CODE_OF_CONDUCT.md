# Contributor Covenant 行为准则

## 我们的承诺

身为项目成员、贡献者、负责人，我们保证为每个人提供一个无骚扰的、开放、热情、积极的协作体验，不论其年龄、体型、残疾、族裔、性别认同和表达、经验水平、国籍、个人形象、种族、宗教信仰、性取向如何。

我们承诺，为每个人创造一个安全、积极的环境，无论是在使用项目的过程中，还是在参与官方社群活动的过程中。

## 我们的准则

有助于创造积极环境的行为包括：

* 使用友好和包容的语言
* 尊重不同的观点和经验
* 优雅地接受建设性的批评
* 关注对社区最有利的事情
* 与其他社区成员表示同理心

不可接受的行为包括：

* 使用与性有关的语言或图像，以及性关注或性挑逗
* 挑衅、侮辱或贬损性评论，以及人身或政治攻击
* 公开或私下骚扰
* 未经明确许可，发布他人的私人信息，如物理地址或电子地址
* 其他可以被合理地认为不适当的行为

## 适用范围

本行为准则适用于本项目的以下场景：

* 项目仓库的 Issues、Pull Requests、Discussions
* 项目的官方社群（如 Discord、微信群等）
* 个人在公开场合代表项目或其社群时

## 执行

项目维护者有权利和责任删除、编辑或拒绝不符合本行为准则的评论（comments）、提交（commits）、代码、Wiki 编辑、议题（issues）和其他贡献。项目维护者可以暂时或永久地封禁任何他们认为行为不当、威胁、冒犯或有害的贡献者。

## 执行

如果发现违反行为准则的行为，请通过以下方式报告：

* **GitHub Issues**: 创建一个标记为 "conduct" 的 Issue
* **Email**: 发送邮件到 gushanaobai@gmail.com

所有投诉都将被审查和调查，并将做出被认为有必要且适当的回应。项目团队有义务对事件报告者保密。

## 执行流程

项目维护者将遵循以下流程来处理违反行为准则的情况：

### 1. 确认收到报告

* 在 24 小时内确认收到报告
* 告知报告者已收到报告并开始调查

### 2. 调查

* 收集相关信息
* 与相关人员沟通
* 查看相关记录（如 GitHub 活动）

### 3. 决定行动

根据调查结果，采取以下一种或多种行动：

* **警告**: 向违规者发出正式警告
* **临时禁言**: 暂时禁止违规者参与项目活动
* **永久封禁**: 永久禁止违规者参与项目活动
* **删除内容**: 删除违规的评论、提交或其他内容

### 4. 通知

* 告知报告者处理结果
* 如果适用，通知违规者处理结果
* 如果适用，通知社区处理结果

### 5. 记录

* 记录事件详情
* 记录采取的行动
* 记录后续跟进

## 报告指南

### 如何报告

如果你看到或经历违反行为准则的行为，请通过以下方式报告：

1. **GitHub Issues**: 创建一个标记为 "conduct" 的 Issue
2. **Email**: 发送邮件到 gushanaobai@gmail.com

### 报告内容

请包含以下信息：

* **事件描述**: 发生了什么？
* **涉及人员**: 谁参与了？
* **时间地点**: 在哪里发生的？什么时候发生的？
* **证据**: 有截图、链接或其他证据吗？
* **影响**: 对你或他人造成了什么影响？

### 报告示例

```
主题: 行为准则违反报告

事件描述:
在 GitHub Issue #123 中，用户 @user1 对用户 @user2 进行了人身攻击。

涉及人员:
- 违规者: @user1
- 受影响者: @user2

时间地点:
- 时间: 2026-06-07
- 地点: GitHub Issue #123

证据:
[截图或链接]

影响:
这使得 @user2 感到不受欢迎，并考虑离开项目。
```

## 社区准则

### 尊重他人

* 尊重每个人的观点和贡献
* 避免使用冒犯性语言
* 接受建设性的批评

### 包容性

* 欢迎不同背景的人
* 使用包容性语言
* 避免偏见和歧视

### 专业性

* 保持专业态度
* 专注于技术讨论
* 避免个人攻击

### 合作精神

* 乐于助人
* 分享知识
* 协作解决问题

## 执行案例

### 警告案例

**情况**: 用户在评论中使用了轻微的冒犯性语言。

**行动**:
1. 删除评论
2. 向用户发出警告
3. 解释行为准则

### 临时禁言案例

**情况**: 用户多次违反行为准则，包括人身攻击和骚扰。

**行动**:
1. 删除违规内容
2. 临时禁言用户 7 天
3. 告知用户禁言原因
4. 如果再次违规，将永久封禁

### 永久封禁案例

**情况**: 用户持续违反行为准则，包括严重骚扰和威胁。

**行动**:
1. 删除所有违规内容
2. 永久封禁用户
3. 报告给 GitHub（如果适用）
4. 通知社区（如果适用）

## 豁免

本行为准则的例外情况需要项目维护者明确批准。在某些情况下，如技术讨论中的激烈辩论，可能会被视为可以接受的行为，但必须保持尊重和专业。

## 修订

本行为准则将定期审查和更新。任何重大更改都将通过以下方式通知社区：

* 在 GitHub Discussions 中发布通知
* 在 README.md 中更新版本信息
* 在 CHANGELOG.md 中记录更改
