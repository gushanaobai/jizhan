## 版本说明

### 版本号规则

本项目使用语义化版本控制：

- **主版本号 (MAJOR)**: 当你做了不兼容的 API 修改
- **次版本号 (MINOR)**: 当你做了向下兼容的功能性新增
- **修订号 (PATCH)**: 当你做了向下兼容的问题修正

### 版本号示例

- `1.0.0`: 第一个正式版本
- `1.1.0`: 添加新功能
- `1.1.1`: 修复 Bug
- `2.0.0`: 重大更新，可能有破坏性变更

### 预发布版本

- `1.0.0-alpha.1`: Alpha 版本
- `1.0.0-beta.1`: Beta 版本
- `1.0.0-rc.1`: 候选版本

### 版本发布周期

- **主版本**: 每 6-12 个月
- **次版本**: 每 1-3 个月
- **修订版**: 根据需要随时发布

### 版本发布流程

1. 创建发布分支 (`release/x.y.z`)
2. 更新版本号
3. 更新 CHANGELOG.md
4. 运行完整测试
5. 创建 Git Tag
6. 构建发布版本
7. 发布到 GitHub Releases
8. 合并到 main 分支
